What we demonstrated (Vulnerable)

This project intentionally demonstrates common web vulnerabilities, then compares them with secure fixes.

1) Stored XSS (Comments)

  • Exploit: In VULNERABLE mode, comments are rendered using innerHTML.
  • Payload: <img src=x onerror=alert('XSS')>
  • Impact: attacker can execute JavaScript on other users.
  • Fix: Secure mode uses textContent / DOM text nodes.

2) Score Tampering (Client-side trust)

  • Exploit: edit mg_scores in LocalStorage and set score to 99999.
  • Impact: leaderboard integrity is broken.
  • Fix: backend must validate or compute score; never trust client values.

3) Identity / Session Tampering

  • Exploit: edit mg_session in LocalStorage and set user to “Admin”.
  • Impact: impersonation + fake scores/comments.
  • Fix: server-side sessions with HttpOnly cookies + authorization checks.

How to run the demo

  1. Go to the game page and play once.
  2. Try XSS payload in VULNERABLE comments mode.
  3. Open DevTools → Application → Local Storage → edit mg_scores.
  4. Edit mg_session to impersonate a user.
  5. Use Reset Demo to clean everything between demos.

Secure version plan (after backend link)

  • Use server sessions (HttpOnly cookies).
  • Authorize score/comment actions using session userId only.
  • Validate and/or compute score on the backend.
  • Sanitize output (or render as text) for comments.
